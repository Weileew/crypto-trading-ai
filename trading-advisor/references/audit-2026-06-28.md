## Root Cause Analysis

**Primary (C2)**: `signal_validator._get()` parameter ordering bug — the params dict passed as second positional argument goes to `headers` instead of `params`. This breaks ALL candle fetching → validation loop dead (C3).

**Secondary (C1)**: CoinGecko free tier rate limits on Phase 1 pagination → non-deterministic market data → briefings show "no candidates" randomly.

**Tertiary (C4)**: Briefing format changed to compact style (bullet format works) but full briefing still uses old `\|` format → parser mismatch on full briefings.
