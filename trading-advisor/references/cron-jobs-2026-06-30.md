## Job Details

### trading-advisor Briefings (Morning/Afternoon)
**Command:** `python3 scripts/briefing.py --compact --enhanced --orchestrator`
- `--compact`: Card-style layout with summary bar, risk, regime, opportunities, watch levels, action
- `--enhanced`: Adds compact regime block (DeFiLlama + DEX + stablecoin flows)
- `--orchestrator`: Appends Orchestrator Status block (params, performance, open signals)

### Orchestrator Nightly
**Command:** Full orchestrator pipeline via `orchestrator.py`
Phases: News → Briefing → Signal Journal → Validation → Performance → Parameter Adaptation → Digest

### Weekly Performance Review
**Prompt:** Concise week-over-week trend from journal.db — win rate, best/worst trades, drawdown, F&G, optimizer recommendations (~200 words, compact bullets)

### Paper Trading M2M (no_agent)
**Script:** `paper-m2m-update.sh` → `python3 scripts/paper_trader.py --update --summary`
- Silent unless position closed (stop/target/trailing) or error
- Delivers card-style notification: `🟢 SYM CLOSED ▲ PROFIT` / `🔴 SYM CLOSED ▼ LOSS`
- Shows open positions + portfolio equity/cash/return

### Research Enrichment
**Command:** `python3 scripts/collect_papers_openalex.py` → `python3 scripts/generate_research_digest.py`
- Runs 05:30 UTC+7 (before 08:00 morning briefing)
- Dual-source: OpenAlex (peer-reviewed) + arXiv q-fin (preprints)
- Regenerates `references/research-digest.md` with Actionable Findings Index

### Parameter Optimizer Weekly

### Crypto Backup Refresh (no_agent)
**Script:** `backup-crypto-repo.sh` — copies Hermes skills to `~/crypto-trading-ai` repo for git backup

## Cron Drift Guard (#44585)

**Problem:** Jobs created with implicit global provider/model snapshot. If global config changes (e.g., switching models), cron safety blocks with: `Skipped to prevent unintended spend: global inference config drifted since this job was created`

**Fix:** Update each job with explicit provider/model pin matching current global config:
```bash
# Get current active model
hermes model

# Update all TOK jobs (example with current model)
for job_id in f0fc8b054fc8 ccc18cada9a7 58e8183595d8 b366c4096b1b af1b44ab18eb 49dfd654806a; do
  cronjob action=update job_id=$job_id model='{"provider":"nous","model":"stepfun-3.7-flash:free"}'
done
```

## Paper Trading Data Sweep Path Fix (2026-06-30)

**Issue:** `paper-trading-audit-reconciliation/scripts/paper_data_sweep.py` used `SKILL_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))` which resolved to the sub-skill directory (`paper-trading-audit-reconciliation`), but data files live in parent `trading-advisor` skill directory.

**Fix Applied:**
```python
# Before (broken):
SKILL_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
REPORTS = os.path.join(SKILL_DIR, "reports")
PAPER = os.path.join(REPORTS, "paper_trading")

TRADING_ADVISOR_DIR = os.path.dirname(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))

PAPER = os.path.join(REPORTS, "paper_trading")

```
